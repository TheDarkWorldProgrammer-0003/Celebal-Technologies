Git Command Cheat Sheet

1. Basic Workflow

git add .
git commit -m "Commit message"
Stages all changed files and commits them with your message

2. Branch Management

git checkout -b new-branch-name
# Make changes
git add .
git commit -m "Changes"
git push origin new-branch-name
Creates a new branch, makes changes, commits them, and pushes to remote

3. Fixing Mistakes

Commits to wrong branch:

git checkout correct-branch
git cherry-pick commit-hash
git checkout wrong-branch
git reset --hard HEAD~1

Restore deleted file:

git checkout HEAD~1 -- path/to/file

Revert to previous commit:

git reset --hard commit-hash

4. Working with Remotes

Push new branch:

git push -u origin branch-name

Open-source contribution flow:

  1. Fork the repository on GitHub
  2. Clone your fork locally: git clone your-repo-url
  3. Create feature branch: git checkout -b feature
  4. Make changes and commit
  5. Push to your fork: git push origin feature
  6. Create pull request from GitHub UI

5. Merge Conflict Resolution

git merge other-branch
# Fix conflicts in files (marked by <<<<)
git add resolved-files
git commit -m "Merge message"
